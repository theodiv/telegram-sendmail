# Code of Conduct

## Purpose

This Code of Conduct establishes the behavioural expectations for all
participants in the `telegram-sendmail` project. It applies to every
interaction space managed by the project, including the issue tracker,
pull requests, code review comments, and commit messages.

## Expected Conduct

The following standards apply to all project interactions:

- Communicate with professionalism and respect, even during disagreements.
- Provide constructive, specific, and actionable feedback in code reviews.
- Accept constructive criticism gracefully and respond to it on its
  technical merits.
- Credit prior work and acknowledge contributions from others.
- Focus discussion on the technical substance of the contribution, not on
  the contributor.

## Unacceptable Conduct

The following behaviours are not tolerated:

- Harassment, intimidation, or personal attacks of any kind.
- Dismissive, demeaning, or condescending language directed at individuals
  or their work.
- Publishing private information (email addresses, physical addresses,
  credentials) without explicit consent.
- Sustained disruption of discussion threads or review processes.
- Any conduct that would reasonably be considered inappropriate in a
  professional setting.

## Scope

This policy applies to all interactions within the project's GitHub
repository, including issues, pull requests, code review comments,
discussions, and commit messages. It also applies when an individual
is representing the project in external public spaces.

## Enforcement

The project maintainer is responsible for interpreting and enforcing this
Code of Conduct. Responses to violations are at the maintainer's discretion
and may include:

- A private warning identifying the specific violation.
- Editing or removing comments, commits, issues, or other contributions
  that violate this policy.
- Temporary or permanent restriction from project interaction spaces.

## Reporting

Conduct concerns should be reported privately by contacting the maintainer
through the contact information on the
[GitHub profile](https://github.com/theodiv). Reports are treated
confidentially. The maintainer will acknowledge receipt, assess the
situation, and respond with an appropriate course of action.
